# Mobile11 Native App Integration Guide (Cloud Build Edition)

This guide explains how to take your existing Lovable web project, merge the native Capacitor files generated by Manus, and use **Codemagic CI/CD** to build and submit your iOS and Android apps entirely in the cloud — **no Mac required**.

---

## Phase 1: Connect Lovable to GitHub

Before you can build the native apps, your Lovable code needs to live in a GitHub repository.

1. **Create a GitHub Account**: If you don't have one, sign up at [github.com](https://github.com).
2. **Connect Lovable**: Open your Mobile11 project in Lovable. Click the **GitHub** icon in the top right (or go to Project Settings → GitHub).
3. **Push to a New Repo**: Follow the prompts to create a new private repository (e.g., `mobile11-app`) and push your Lovable code to it.

---

## Phase 2: Merge the Native Files

Now you need to merge the native files provided in this zip archive into your new GitHub repository.

### Option A: Using the provided script (Recommended)

If you have a terminal (Mac, Linux, or Windows WSL):

1. Clone your new GitHub repo to your computer:
   ```bash
   git clone https://github.com/your-username/mobile11-app.git
   ```
2. Extract the `mobile11-app.zip` file provided by Manus.
3. Run the merge script:
   ```bash
   bash mobile11-app/scripts/merge-native-into-lovable.sh ./mobile11-app ./mobile11-app-extracted
   ```

### Option B: Manual Copy

If you prefer to do it manually, copy these folders and files from the extracted zip into your Lovable repo:
- `ios/` (Entire folder)
- `android/` (Entire folder)
- `native-plugins/` (Entire folder)
- `resources/` (Entire folder)
- `src/lib/*.ts` (Copy all TypeScript files into your existing `src/lib/` folder)
- `public/.well-known/` (Copy the folder into your existing `public/` folder)
- `capacitor.config.ts`
- `codemagic.yaml`

## Phase 3: Update Your Lovable Code

You need to make a few small changes to your Lovable codebase to initialize the native plugins.

### 1. Install Dependencies

Open your terminal in your Lovable repo and run:

```bash
npm install @capacitor/core @capacitor/cli @capacitor/ios @capacitor/android
npm install @capacitor/app @capacitor/browser @capacitor/preferences
npm install @capacitor/push-notifications @capacitor/haptics @capacitor/status-bar
npm install @capacitor/splash-screen @capacitor/device @capacitor/network
npm install @capacitor-community/stripe @aparajita/capacitor-biometric-auth
npm install @capacitor/live-updates
```

### 2. Initialize Native Plugins in `App.tsx`

Open your `src/App.tsx` file and add the `NativeInitializer` component inside your main router/provider tree.

```tsx
import { useEffect } from 'react';
import { initializeDeepLinks } from './lib/deepLinks';
import { initializePushNotifications } from './lib/nativePush';
import { SplashScreen } from '@capacitor/splash-screen';
import { isNativePlatform } from './lib/platform';

// Add this component
function NativeInitializer() {
  useEffect(() => {
    if (!isNativePlatform()) return;

    const init = async () => {
      try {
        await initializeDeepLinks();
        await initializePushNotifications();
        await SplashScreen.hide();
      } catch (e) {
        console.error('Native init error:', e);
      }
    };
    init();
  }, []);
  return null;
}

// Inside your App component's return statement, add <NativeInitializer />
function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <TooltipProvider>
        <Toaster />
        <NativeInitializer /> {/* <-- Add this here */}
        <RouterProvider router={router} />
      </TooltipProvider>
    </QueryClientProvider>
  );
}
```

### 3. Commit and Push

```bash
git add .
git commit -m "Add native iOS and Android Capacitor shell"
git push origin main
```

## Phase 4: Set Up Developer Accounts

While you don't need a Mac, you still need developer accounts to publish apps.

1. **Apple Developer Program**: Sign up at [developer.apple.com](https://developer.apple.com) ($99/year). This can take 24-48 hours to activate.
2. **Google Play Console**: Sign up at [play.google.com/console](https://play.google.com/console) ($25 one-time fee).

---

## Phase 5: Configure Codemagic CI/CD

Codemagic will act as your "Mac in the cloud" to build the iOS and Android apps.

1. **Sign Up**: Go to [codemagic.io](https://codemagic.io) and sign up with your GitHub account.
2. **Add Application**: Click "Add application" and select your Mobile11 GitHub repository.
3. **Select Project Type**: Choose **Ionic Capacitor**.
4. **Configure iOS Code Signing**:
   - Go to your App settings in Codemagic → **Code signing identities** → **iOS certificates**.
   - Connect your Apple Developer account (App Store Connect API key).
   - Codemagic will automatically fetch your certificates and provisioning profiles.
5. **Configure Android Code Signing**:
   - Go to **Android keystores**.
   - Generate a new keystore or upload an existing one.
   - Name the reference `keystore_reference` (this matches the `codemagic.yaml` file).
6. **Google Play Integration**:
   - Create a Google Cloud Service Account with publishing rights to Google Play.
   - Upload the JSON key to Codemagic under **Google Play** integration.
7. **Update `codemagic.yaml`**:
   - Open `codemagic.yaml` in your repo.
   - Replace `APP_STORE_APP_ID: 0000000000` with your actual App ID from App Store Connect.
   - Replace `nico@mobile11.com` with your actual email address.
   - Commit and push the changes.

---

## Phase 6: Build and Submit

### 1. Trigger a Build

In Codemagic, click **Start new build**. Select the `ios-release` or `android-release` workflow defined in your `codemagic.yaml`.

Codemagic will spin up a Mac Mini M2 in the cloud, install your dependencies, build the web app, sync it to the native iOS/Android projects, compile the native binaries (`.ipa` and `.aab`), sign them, and upload them directly to TestFlight and Google Play Internal Testing.

### 2. Test on Real Devices

- **iOS**: Open the **TestFlight** app on your iPhone and download the Mobile11 app.
- **Android**: Open the opt-in link from Google Play Console on your Android phone and download the app.

Test the eSIM installation, Apple Pay/Google Pay, and push notifications.

### 3. Submit for Review

Once you are happy with the TestFlight/Internal Testing builds:

- **Apple App Store**: Go to App Store Connect, select the build, fill out the metadata (screenshots, description), and click **Submit for Review**.
  - *Important*: In the review notes, paste the IAP-exemption text regarding Apple guideline 3.1.3(e) (consumable goods used outside the app).
  - Provide a demo account (`apple-review@mobile11.com`) with a pre-loaded test eSIM.
- **Google Play**: Go to Google Play Console, promote the Internal Testing release to Production, fill out the Data Safety form, and submit.

## Phase 7: The Magic of Live Updates

After your app is live in the stores, your day-to-day workflow becomes incredibly simple:

1. Make changes in **Lovable** (UI tweaks, new plans, pricing changes).
2. Click **Publish** in Lovable.
3. The web app updates instantly.
4. The native apps (iOS and Android) will automatically download the new web bundle in the background via `@capacitor/live-updates` the next time the user opens the app.

**You only need to trigger a new Codemagic build and resubmit to the stores when you:**
- Change the app icon or splash screen.
- Add a new native capability (e.g., Camera, NFC) that requires new permissions.
- Update the native eSIM installer plugin or Stripe SDK.
- Need to comply with annual iOS/Android SDK version bumps required by Apple/Google.
